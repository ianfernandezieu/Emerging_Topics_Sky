1 Project Overview

This project forecasts airport congestion pressure at Adolfo Suarez Madrid-Barajas (LEMD), Spain's busiest airport. We built a machine learning pipeline that ingests daily IFR flight movements from Eurocontrol, enriches them with weather data and calendar signals, computes a composite congestion score (ACPS), and trains gradient-boosted tree models for both regression and classification.

Key question: Can we forecast airport congestion pressure using flight volume, weather, and calendar signals?
Answer: Yes -- our model achieves R² = 0.96 on unseen test data, accurately predicting daily congestion with a mean absolute error of just 0.45 points on a 0--100 scale.

The original prototype used ~25 hourly records from FlightRadar24 scraping. This new pipeline uses 3,318 daily records from the official Eurocontrol IFR traffic dataset, covering January 2017 through February 2026 -- a 130x increase in data volume.

2 Data Sources

E

Eurocontrol IFR Daily Traffic

Airport_Traffic.xlsx (DATA sheet) -- 1,021,929 rows across all European airports. Filtered to LEMD: 3,346 daily records with IFR departures, arrivals, and total movements. Network Manager (NM) figures used (zero nulls vs 29 nulls in airport-reported figures).

W

Open-Meteo Historical Weather

Free API providing hourly weather for Madrid-Barajas coordinates (40.47N, -3.57W). 10 variables fetched: temperature, humidity, precipitation, rain, weather code, surface pressure, wind speed/direction/gusts, cloud cover. Aggregated from hourly to daily (means, maxes, sums).

H

Nager.Date Public Holidays

Spanish public holidays for 2017--2026 (99 dates total). Used to build holiday, pre-holiday, post-holiday, and bridge-day indicator features that capture demand shifts around non-working days.

Step 1: Load & Filter Eurocontrol Data

The Eurocontrol Airport_Traffic.xlsx file contains daily IFR movement counts for every major European airport since 2017. We read the DATA sheet (1,021,929 rows) and filter to APT_ICAO == "LEMD", yielding 3,346 daily records. We use the Network Manager (NM) columns (FLT_DEP_1, FLT_ARR_1, FLT_TOT_1) which have zero null values, rather than the airport-reported IFR figures (29 nulls).

Step 2: Fetch Weather Data

Hourly weather is fetched from the Open-Meteo Historical Weather API for the Madrid-Barajas coordinates. The API has a ~1-year limit per request, so we chunk the 9-year range into yearly segments. The 10 raw hourly variables are aggregated to daily:

  • Mean: temperature, humidity, wind speed, pressure, cloud cover, wind direction
  • Sum: precipitation, rain
  • Max: weather code (severity), wind gusts, peak wind speed

Engineered features are added: sin/cos wind direction encoding, severe weather flag (WMO code ≥ 65), and rain indicator. All weather data is cached to parquet after first fetch.

Step 3: Fetch Holidays

Spanish public holidays for each year (2017--2026) are fetched from the Nager.Date API. Only "global" holidays (national, not regional) are included. Each year's JSON response is cached locally. Total: 99 holiday dates.

Step 4: Calendar Features

From each date, we extract 15 features:

  • Temporal: day of week (0-6), month (1-12), quarter, day of year
  • Binary: is_weekend, is_holiday, is_pre_holiday, is_post_holiday, is_bridge_day
  • Cyclic: sin/cos encodings of day-of-week, month, and day-of-year (to capture periodicity without artificial boundaries)

Step 5: Build Model Table

All sources are merged on date. The ACPS target (Airport Congestion Pressure Score, 0--100 scale) is computed, along with lag features, rolling windows, and additional derived features. See Section 5 for details.

Step 6: Chronological Split

The data is split chronologically (no shuffling, to prevent data leakage from the future):

Train: 2,322 (Jan 2017 -- Jun 2023)
Valid: 498
Test: 498

Step 7: Train & Evaluate

A HistGradientBoostingRegressor predicts continuous ACPS. A HistGradientBoostingClassifier predicts congestion level (Low/Medium/High). Both use max_iter=200, max_depth=6, learning_rate=0.1. Evaluation uses MAE, RMSE, R² for regression and accuracy/F1/balanced accuracy for classification.

ACPS Target Computation

The Airport Congestion Pressure Score is a composite metric:

ACPS = rescale_to_0_100(0.6 * z(total_movements) + 0.4 * z(pressure_ratio)) where: z(x) = (x - mean(x)) / std(x) # z-score standardization pressure_ratio = movements / median(movements for same day-of-week) rescale(x) = (x - min) / (max - min) * 100 # normalize to 0-100 scale

The 60/40 weighting means raw traffic volume (movements) is the primary driver, while the pressure ratio captures whether the airport is busier than typical for that day of the week. The z-score standardization ensures both components contribute proportionally regardless of their raw scales.

Congestion Classification

ACPS is discretized into three classes based on percentile thresholds:

  • Low ACPS below 60th percentile -- normal operations
  • Medium ACPS between 60th and 85th percentile -- elevated pressure
  • High ACPS above 85th percentile -- significant congestion risk

Weather Correlation with ACPS

Pearson Correlation: Weather Variables vs ACPS

Weather has weak direct correlation with ACPS at the daily level. Temperature shows the strongest relationship (r = 0.11), likely as a proxy for season (summer = more flights). Weather's value is more in capturing edge cases: severe storms or heavy rain days that cause operational disruptions.

6 Model Training

Algorithm: HistGradientBoosting

We use scikit-learn's HistGradientBoostingRegressor and HistGradientBoostingClassifier, which are histogram-based gradient boosting implementations similar to LightGBM. Key advantages:

  • Native handling of missing values (NaN in lag_365d features)
  • Fast training on medium-sized tabular data
  • Built-in regularization through max_depth and min_samples_leaf
  • No feature scaling required

9 Feature Importance

Feature importance is computed via permutation importance on the test set: each feature is randomly shuffled and the resulting performance degradation is measured. Higher values mean the feature is more critical for accurate predictions.

Top 15 Features by Permutation Importance

Interpretation

  • total_movements (0.90): Overwhelmingly dominant -- raw traffic volume is the strongest predictor of congestion, as expected. This validates that ACPS correctly reflects operational load.
  • departures (0.08) & arrivals (0.03): The split between departures and arrivals captures directional imbalance effects. Departure-heavy days have different congestion characteristics than arrival-heavy days.
  • dow_sin (0.02) & dow_cos (0.01): Day-of-week cyclic encoding captures the weekly periodicity. Saturday's lower traffic and Friday's peak are learned patterns.
  • acps_rmean_7d (0.016): The 7-day rolling average provides trend context -- is congestion rising or falling week-over-week?
  • movements_lag_365d (0.003): Year-over-year comparison, useful for capturing long-term growth trends and the COVID recovery trajectory.
  • Weather features: Low individual importance but collectively contribute to edge-case accuracy. Their value lies in explaining outlier days (storms, extreme heat) rather than driving typical predictions.
